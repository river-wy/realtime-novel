## 1. 方向调整（v0.2 → v0.3）

### 1.1 核心判断（欧尼酱 2026-06-15 11:48 对齐）

> "M1 先这样，算法我们后面再详细调整。现在我想快速往下推进，先把整个产品的全链路骨架搭建起来。细节算法可以后面慢慢调整。"

### 1.2 调整前后对比

| 维度 | 调整前（05 路线图主线） | 调整后（本文件主线）|
|------|----------------------|---------------------|
| **首要目标** | M1 算法调优 → 5 指标真实 baseline | **全链路骨架搭建** → 产品可运行 |
| **代码产出** | `eval/v0.3/` 评测脚本 | **`realtime_novel/` 产品骨架包** |
| **度量标准** | 5 指标达标率 | **骨架可加载 + 端到端跑通** |
| **时间投入** | 1-2 天调参 | **1-2 天立骨架（S1+S2）** |
| **算法** | 立刻调 `planned_interval` | **M-α 阶段不调**，原值沿用 |
| **并行关系** | M1-M4 串行 | **骨架与算法并行**：骨架先成型，调优在骨架上做 |

### 1.3 关键原则

1. **骨架优先** —— 没有可用的产品代码，再准的算法调优也只是评测玩具
2. **算法沿用** —— v0.2 的 4 维权重、planned_interval、sigmoid k 值全部沿用
3. **接口先行** —— S1-S5 先定接口，算法层换实现不影响产品侧
4. **可回滚** —— 每个骨架版本的产物 + 代码都落盘，不破坏 v0.2 评测
5. **配置集中** —— 复用 v0.2 LLM 配置（`~/AiTest/lunaris/config.private.json`），不重搭

### S2 · WorldTree 数据结构

**职责**：Node / 主线 / 支线 / 种子 / 摘要 的内存模型 + YAML 序列化。

```python
class WorldTree:
    main_plot: MainPlot              # 主线（树干）
    branches: Dict[str, Branch]      # 支线（分枝）
    nodes: List[Node]                # 已生成节点（按时间序）
    seeds: SeedBook                  # 种子库
    summaries: SummaryChain          # 摘要链
    
    def to_dict(self) -> dict: ...
    @classmethod
    def from_dict(cls, data: dict) -> "WorldTree": ...
    
    def add_node(self, node: Node) -> None: ...
    def rollback_to(self, node_id: str) -> None:
        """硬 reset：删除该节点之后的所有内容"""
```

**关键设计点**（来自 `01 §1.4`）：
- **单线树** —— 永远只保留一条主干
- **回档硬 reset** —— 从回档点起，原先生成的所有枝叶全部删除
- **Node 保留** —— 回档点之前的 Node 仍可回看

**依赖**：
- `01-world-contract.md` §1 树形结构 + 生长机制
- `03-product-schemas.md` 各 YAML 格式

### S4 · 章节生成 orchestrator

**职责**：输入干预 → 组装 prompt → 调 LLM → 写章节 → 更新 WorldTree。

```python
class ChapterGenerator:
    def generate(self, 
                 tree: WorldTree, 
                 intervention: Intervention) -> Chapter:
        """生成下一章节"""
        # 1. 上下文组装（种子 + 摘要 + 干预 + 风格锚点）
        # 2. Prompt 构造（3 层 prompt）
        # 3. LLM 调用（复用 v0.2 LLM 客户端）
        # 4. 章节文本解析 + 字数校验（≥ 3000）
        # 5. 提取种子 / 更新摘要 / 写入 WorldTree
        # 6. 落盘 chapters/chapter-NN.txt
```

**关键设计点**：
- **3 层 prompt** —— 来自 `01-03` 各处定义
- **种子提取** —— 生成后从章节文本提取新种子，写入 seeds.yaml
- **摘要滚动** —— 每 15 章生成一次阶段摘要（来自 `02 §3`）
- **字数校验** —— `00 §2.1` 要求 ≥ 3000 字

**依赖**：
- S2（操作 WorldTree）
- L3 LLM 客户端（v0.2 复用）
- `01-03` 各处 prompt 设计

### S5 · 干预 + 回档 orchestrator

**职责**：导演/演员输入解析 + 回档硬 reset。

```python
class InterventionParser:
    def parse(self, raw: str, mode: Literal["director", "actor"]) -> Intervention:
        """解析用户输入为结构化干预"""
        # director: "我期望 xxx" → 引导式
        # actor:    "我 xxx"      → 入戏式

class RollbackManager:
    def rollback(self, tree: WorldTree, target_node_id: str) -> None:
        """硬 reset 到指定 Node"""
        # 1. 校验 target_node_id 存在
        # 2. 删除 target_node 之后的所有 nodes
        # 3. 清空对应的种子 / 摘要
        # 4. 备份被删内容到 .trash/（可恢复保险）
        # 5. 更新 current_node
```

**关键设计点**（来自 `00 §2.2` + `01 §1.4`）：
- **双视角双干预语义** —— 第三视角是引导、第一视角是入戏
- **回档不破坏历史 Node** —— Node 仍可回看
- **保险箱 `.trash/`** —— 实际产品考虑，先不实现（M-α 简化版）

**依赖**：
- S2（操作 WorldTree）
- L1 持久化（文件清理）
