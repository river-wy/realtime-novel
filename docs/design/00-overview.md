## 1. 背景与动机

### 1.1 用户痛点（欧尼酱原话，2026-06-11）

- 小说平台**套路化严重**，内容、主题千篇一律
- 很多主题不喜欢
- **缺少沉浸感**
- **内容套路化**（不够生动灵活）

### 1.2 为什么是「实时生成 + 可干预」

传统小说的问题不在于 AI 写得差，而在于**作者和读者的权力不对等**——用户只能被动接受。

实时生成 + 可干预把这种权力结构打破：

- **套路化** → 实时生成没有预设模板
- **价值观同质化** → 用户主动塑造
- **节奏被动** → 用户主动控制
- **沉浸感缺失** → 用户从读者变成参与者

## 2. 核心产品定义

### 2.1 产品一句话

> **一「章」一「生成」，双视角切换，导演/演员两栖的实时小说阅读体验。**

### 2.2 关键概念

| 概念                     | 定义                          | 备注             |
|------------------------|-----------------------------|----------------|
| **章（chapter / round）** | 一次 LLM 生成输出的内容单元，至少 3000 字  | 不是传统小说章节，是「一轮」 |
| **第三视角（导演模式）**         | 用户读全知视角故事，干预方式：「我期望 xxx」    | 引导式输入          |
| **第一沉浸视角（演员模式）**       | 用户代入角色，干预方式：「我 xxx」         | 入戏式输入          |
| **风格锚点**               | 表达用户偏好的机制（标签 / 自由文本 / 参考作者） | 详细设计待展开        |
| **多 agent 叙事（未来）**     | 主线旁白 agent + 人物 agent 协作    | 第一版不做          |

### 2.3 与已有形态的对比

| 形态                      | 关系  | 核心差异                   |
|-------------------------|-----|------------------------|
| 传统小说阅读                  | 不替代 | 实时生成，不是已完成作品           |
| AI 写完整本小说               | 不替代 | 用户不是旁观者，是共同创作者         |
| Galgame                 | 有交集 | Galgame 是预制剧本，本项目是实时生成 |
| TRPG / 跑团               | 有交集 | 跑团重交互轻叙事，本项目重叙事轻交互     |
| 互动小说（如 Life is Strange） | 有交集 | 互动小说是预制分支树，本项目是实时生成    |

## 3. 已对齐的产品原则

1. **「章」是核心产品单元**，不是章节数，是一次输出单位
2. **3000 字起步**——少了没读的意义
3. **双视角双干预语义**——第三视角是引导、第一视角是入戏
4. **沉浸感优先于反套路**——沉浸感是 P0，反套路是 P1
5. **第一版单人**——多 agent 是后续扩展
6. **不抢跑**——方案没对齐不写代码
7. **讨论节奏**——按"对齐一个阶段 → 写一个文件"推进

### 2026-06-11 · 17:00-17:30 · 沉浸感路径 - 测评对齐（webchat）

**议题**：沉浸感 = 什么？怎么量化？

**方法**：蕾姆酱抛 5 个不同题材/风格/密度配比的"场景切片"（每片 300-500 字），欧尼酱给"沉浸感 0-10 分"+"那一笔"+"还想"。

**测评结果**：

| 切片    | 题材      | 配比（情/信/行） | 评分     | 欧尼酱的"那一笔"                       |
|-------|---------|-----------|--------|---------------------------------|
| **A** | 日常 / 拥吻 | 情感优先      | **10** | "呼吸停在嗓子眼/手指卷袖口"+"别动...就看看你会不会躲" |
| **B** | 心理惊悚    | 信息优先      | **7**  | "吸鼻子 / 戒烟后那种吸法"（想加可感知声音 + 身体印记） |
| **C** | 修真 / 战斗 | 行动优先      | **8**  | "天际再亮一点就被认出 / 0.3 秒"            |
| **D** | 玄幻 / 背叛 | 情+信双密度    | **10** | "那……之后呢？"（递进式披露）                |
| **E** | 民国 / 送别 | 留白        | **6**  | （散文风不感冒，反例）                     |

**提炼的洞察**：

1. **"那一笔"的共性 = 具体性（specificity）**
    - 不是"写满细节"，是"这个细节只有这一刻、这一角色、这一个人才能有"
    - 反例：可被替换到任何小说的细节 = 不是"那一笔"

2. **4 种具体性颗粒类型**（从 5 个评分案例反推）
    - **身体微动作**：哪块肌肉、哪个部位、什么微反应（A：呼吸/手指）
    - **私密度记忆**：只有角色本人知道的"只有他懂"的小知识（B：戒烟吸法）
    - **精确到不可能虚构**：用具体数字/物理量逼出真实感（C：0.3 秒）
    - **节奏与标点**：省略号 / 短句 / 留白的位置（D：那……之后呢？）

3. **D 拿 10 分的意外** — 信息密度（递进披露）+ 情感锚点（17年养育）叠加 = 双重沉浸。**信息密度和情感密度不是平行关系，是叠加的
   **。

4. **欧尼酱的"还想"** — 提示要更**细节**（反应/动作/场景变化）+ 要**可感知声音/触感/内心** + **对散文/留白耐受度低**。

5. **E 是最有价值的负样本** — 单一留白 + 缺乏具体性密度 = 平。**欧尼酱偏好"密度路线"而非"克制路线"**。

**核心结论（暂定，待反例测试 F/G 验证）**：

> **沉浸感 = 具体性 + 密度（两者独立必要）**
> "具体性"维度：4 种颗粒类型
> "密度"维度：情/信/行三维，按场景类型配比

### 2026-06-11 · 18:00-18:30 · 反例测评 + 模型修正（webchat）

**议题**：验证"具体性 × 密度"是否真的是两个独立必要条件，得出 v2→v3 修正。

**方法**：蕾姆酱抛 5 个反例切片（F-J），**不告诉欧尼酱哪个是正面哪个是反面**，凭真实感觉打。

**测评结果**：

| 切片    | 题材        | 第一轮同型对照     | 评分    | 欧尼酱的"那一笔"                   | 欧尼酱的"还想"         |
|-------|-----------|-------------|-------|-----------------------------|------------------|
| **F** | 仙侠 / 师徒决裂 | D（10）/ C（8） | **8** | "半边脸鹤发 / 半边脸少年 / 你不该看到那个铭文" | 缺情绪连贯性，不知怎么代入女主  |
| **G** | 民国 / 暗杀   | C（8）        | **8** | "血珠从唇角往下走"                  | 14岁那段逻辑不顺；缺自白/心理 |
| **H** | 校园 / 高考前夜 | A（10）/ E（6） | **6** | （没抓人的）                      | 题材不感冒            |
| **I** | 古风 / 大婚之夜 | D（10）       | **9** | "袖口线头比寝衣还白"                 | 缺动作互动（"他怎么走过来"）  |
| **J** | 都市 / 离婚前夜 | E（6）        | **8** | "飞机 / 律师函方块 / 戒指"           | 不知该代入谁/什么情感      |

**三个新洞察**：

1. **"逻辑连贯"和"代入感"是两件不同的事**
    - F 缺叙事链/因果链
    - J 缺情绪路标
    - → 产品原则 1：**场景内要"情绪指向明确" + "信息流连贯"双指标**

2. **题材共鸣是独立维度（乘子而非加项）**
    - H 6 分不是写得差，是题材不感冒
    - → 产品原则 2：风格锚点首先解决"我不爱什么"，再谈"我喜欢什么"

3. **主观性 = 独立维度（不再藏身于"具体性"里）**
    - 4 个"还想"全部指向"想看到某个视角下的内心/动作"
    - → 产品原则 3：场景内要"认知窗口明确" + "情绪路标清晰"

**v2→v3 模型升级**：

```
v2：沉浸感 = 具体性 × 密度
v3：沉浸感 = 具体性 × 主观性 × 密度 × 题材共鸣
                 └─新增──┘  └─新增─/
```

**修正项对照**：

| v2 判断          | 测评后修正                               |

| "欧尼酱对留白零耐受"    | 错。单点留白才是问题——J = 8 分证明留白+具体性颗粒托底可拿 8 |
| "具体性 × 密度"充分必要 | 不充分，加了主观性和题材共鸣两个独立必要条件              |

### 2026-06-11 · 19:45-20:08 · 第三轮长切片 + 模型修正（webchat）

**议题**：验证 v3 在长篇场景下是否稳定，识别 v3 漏掉的边界。

**方法**：蕾姆酱抛 5 个 1500-2000 字长切片（K/L/M/N/O），**不告诉欧尼酱设计意图**。O 是微调教向（按欧尼酱要求重写）。

**结果**：**欧尼酱没打具体分数**——原话"片段化的内容缺少上下文，不好客观给内容打分"。
**但给出了两条关键体感反馈**：

1. **细节过度**（心理活动描述过度，如 L 里面太多次"算过""记得"）
2. **数字应用过度**（少量关键数字是亮点，大量使用就是冗余）

**蕾姆酱的诊断**：

- v3 写"汉尼拔式具体性过载"——**这句口号本身是错的**
- 第三轮我延长了篇幅但**没调节密度**，等于**单章颗粒数 × 3**
- "密度"被错误等同于"数量"——**真正的密度是"对比度"**

**v3 → v3.1 修正**：

- **硬上限**：每千字 ≤ 3 个具体性颗粒（数字/心理/动作/记忆/标点合计）
- **数字颗粒更严**：关键数字 ≤ 3 个 / 章
- **心理活动更严**：核心心理动作 ≤ 2 次 / 场景
- **种子延续原则**（欧尼酱原话）：具体性不是"无限新增"，而是"关键字的延续"
- **跨章种子库**：一章 3000 字 = 9 颗强颗粒配额，但应是 2-3 颗"种子"在 9 个位置复现
- **种子复现 ≠ 重复**：同一颗种子在不同章节里应该**生长出不同含义**

**欧尼酱的 20:06 反馈原文**：

- 「每千字最多不超过 3 个，并且这个具体性密度应该不是无限新增，而是关键字的延续」
- 「还好，当前只是定义方向，后面还可以调整，因为最终还是会体现在实时给模型的 prompt 中，调整成本还好」

**v3.1 终版公式**：
> **沉浸感 = 具体性 × 主观性 × 密度 × 题材共鸣**
> + **具体性颗粒硬上限 ≤ 3 / 千字**
> + **种子颗粒在跨章间复现**（2-3 颗 / 故事）

**v3.1 收口状态**：✅ 冻结（2026-06-11 20:08），进入下一阶段讨论。

## 8. 沉浸感实现路径（v3.1 · 2026-06-11 20:08 收口）

> 本节已从 v1（"密度"）→ v2（"具体性 × 密度"）→ **v3（"具体性 × 主观性 × 密度 × 题材共鸣"）** 演化。
> 来源：第 5 节两轮测评会话（A-E 初始 + F-J 反例 + 测评修正）。

### 8.1 核心定义（v3 终版）

> **沉浸感 = 具体性 × 主观性 × 密度 × 题材共鸣**
> 四个独立必要条件，**全部满足时沉浸感才涌现**。

四个维度的定义：

| 维度       | 含义                                   | 反面（不满足时的样子）            |
|----------|--------------------------------------|------------------------|
| **具体性**  | 细节的"不可替代性"——是**只有这一刻、这一角色、这一个人**才能有的 | "她很紧张"（可被替换到任何小说）      |
| **主观性**  | 细节是**从谁的认知窗口看出去的**——情绪路标是否清晰         | 氛围到位但不知该带什么情感（典型：F）    |
| **密度**   | 在同一场景内，具体性颗粒的**种类和数量**               | 切片 E：单一留白 + 颗粒不足       |
| **题材共鸣** | 用户的题材偏好乘子——H 的 6 分不是因为写得差            | 同一种写法，A 题材得 10，H 题材得 6 |

**关键修正**（v2 → v3）：

- v2 把"主观性"藏在"具体性"里——**错了**。F/G/I/J 的"还想"全部指向"想看到某个视角下的内心/动作"——主观性是**独立维度**
- v2 没把"题材共鸣"纳入模型——**也错了**。H 暴露了"用户共鸣是乘子，不是加项"

### 8.2 具体性颗粒的 4 种类型（v3 保留）

| 类型           | 定义               | 测评中的例子                    | 适配密度    |
|--------------|------------------|---------------------------|---------|
| **身体微动作**    | 哪块肌肉、哪个部位、什么微反应  | A："呼吸停在嗓子眼 / 手指卷紧袖口一小块布料" | 情感      |
| **私密度记忆**    | 只有角色本人知道的小知识     | B："戒烟后才有那种吸法"             | 信息      |
| **精确到不可能虚构** | 具体数字 / 物理量       | C："0.3 秒"                 | 行动      |
| **节奏与标点**    | 省略号 / 短句 / 留白的位置 | D："那……之后呢？"               | 信息 + 情感 |

**重要**：这 4 种是"工具"，**不是"目标"**。一个好的具体性颗粒常常是 ≥2 种类型的叠加。
**J 的修正**：留白本身不是问题（飞机 + 律师函方块 + 戒指 = 3 个具体性颗粒托底，单点密度够 → 8 分）。**单点留白才是问题**。

### 8.3 主观性的 3 个子维度（v3 新增）

| 子维度      | 含义                               | 测评案例                         |
|----------|----------------------------------|------------------------------|
| **认知窗口** | 谁的视角在看？叙述者知道多少？                  | F 缺女主认知窗口 → 8 而非 10          |
| **情绪路标** | 是否清楚告诉用户"该往哪个方向感受"               | J 缺路标 → "不知该代入谁/什么情感"        |
| **互动链**  | 角色 A 的认知 → A 的行动 → B 的反应 → B 的认知 | I 缺男主那侧的认知 / 缺动作互动（"他怎么走过来"） |

**产品意义**：LLM-as-judge 自查清单要加这一轴——"本场景的情绪路标是否清晰"。

### 8.4 密度配比（场景类型 × 情/信/行）

> **不是每章都堆满 3 个密度**。不同场景类型有不同最优配比。

| 场景类型      | 情感    | 信息    | 行动    | 测评对应        | 备注             |
|-----------|-------|-------|-------|-------------|----------------|
| 慢热日常 / 暧昧 | **高** | 中     | 低     | A（10）       | 极精细的感官密度       |
| 心理惊悚 / 独处 | 高     | **高** | 低     | B（7→9?）     | 需补可感知声音 + 身体印记 |
| 修真战斗 / 决战 | 中     | 中     | **高** | C（8）/ G（8）  | 行动密度稳定 8       |
| 真相揭示 / 背叛 | **高** | **高** | 低     | D（10）/ F（8） | 需补主观性才能稳定 10   |
| 散文 / 留白   | —     | —     | —     | E（6）        | 反例证伪"冰山理论"     |
| 民国谍战 / 动作 | 中     | **高** | **高** | G（8）        | 行动+信息双密度       |
| 校园现实 / 心理 | **高** | 中     | 低     | H（6）        | 题材共鸣不足         |
| 古风宫廷 / 暧昧 | **高** | **高** | 低     | I（9）/ A（10） | 同 A 模型但信息维度更重  |
| 都市情感 / 日常 | **高** | 中     | 低     | J（8）        | 留白+具体性颗粒托底可拿 8 |

**关键约束**：

- 双密度（情+信）是高分稳定区：A=10 / D=10 / I=9 / F 缺主观性=8
- 单维度极端（只具体性 或 只密度）会掉分

### 8.5 题材共鸣作为外部乘子（v3 新增）

> 这是 v3 相对 v2 **最大的结构变化**。

**测评证据**：

- H（校园高考题材）= 6 分，欧尼酱自述"故事本身不吸引我"
- 同写法的"具体性 × 主观性"内容，在不同题材下评分天差地别

**产品意义**：

- **"具体性 × 主观性 × 密度"是写作技法层**——决定**写得**好不好
- **"题材共鸣"是用户偏好层**——决定**用户**买不买账
- 后者**先于**前者——用户预表达的"我不爱什么"必须**先**于生成

→ **回响 P0 的"风格锚点"问题**：风格锚点**首先**是"题材共鸣"的过滤器，然后才是"我喜欢/我讨厌"的细节

### 8.6 具体性颗粒的密度上限（v3.1 关键修正）

> ⚠️ **v3 → v3.1 的最重要修正**。第三轮长切片测评证伪了"具体性越多越好"的直觉。

**硬上限**：

- **每千字 ≤ 3 个具体性颗粒**（含数字 / 心理活动 / 身体微动作 / 私密度记忆 / 节奏标点 全部类型合计）
- 数字颗粒单独更严：**关键数字 ≤ 3 个 / 章**
- 心理活动颗粒单独更严：**核心心理动作 ≤ 2 次 / 场景**

**为什么会过载**：

- 第三轮 L 切片的 1500-2000 字里塞了 N 个"算过/记得/数过/量过"心理活动 + 大量"0.3 / 0.7 / 39.5 / 2555 / 0.2 厘米"等数字
- **每个单独看都是好颗粒，但堆在一起变成"内心独白循环"+ 数字标签化**
- 欧尼酱原话："细节过度了 / 数字应用过度"

**核心洞察**：
> **具体性是密度问题，不是总量问题**
> 一章 3000 字里有 **3 颗强颗粒 + 5 颗弱托底** > **20 颗等强颗粒**
> 真正的密度 = **对比度**——3 颗钻石撒在白沙上 > 30 颗钻石撒在钻石堆里

**对生成器的硬指标**：

- 生成器必须能自查："本章我用了几个具体性颗粒？"
- 超出上限的颗粒应该被**降级为弱托底**（不删除，但**降密度**——比如从"算过"变成"记不清"）
- 第一版实现路径：LLM-as-judge 检查"颗粒数/千字"是否 ≤ 3

### 8.7 颗粒的纵向延续原则（v3.1 核心新增）

> **欧尼酱原话**："具体性密度应该不是无限新增，而是**关键字的延续**"

**新规则**：

- **同一颗"种子颗粒"在多章间复现**——不是每章独立发明
- 一章 3000 字 = 9 颗强颗粒配额，但**应该是 2-3 个"种子颗粒"在 9 个位置复现**

**测评证据**（A/D/M/O 切片中的"0.3"）：

| 切片     | 出现的"种子"                   | 复现位置                        |
|--------|---------------------------|-----------------------------|
| A · 拥吻 | 0.3 秒（他手停的时长）             | 介入点前的那一刻                    |
| M · 咖啡 | 0.3 秒（他手停的时长）             | 跟 A 同源——同一个男人的同一动作          |
| O · 离婚 | 0.3 毫米（茶渍深） / 0.3 米（沙发距离） | 数字不变，含义变了（从"心动停顿"到"婚姻裂痕距离"） |

→ **同一个数字，在不同章节的语境里生长出不同含义**——这是"种子"的核心价值

**产品意义**：

- **跨章"种子库"** = 故事的记忆系统
- **单章是横截面密度，跨章是纵向延续**——两件事必须一起做
- 生成器需要**"种子注册表"**：故事开场前规划 2-3 颗种子，每章按节奏复现
- 复现 ≠ 重复：同一颗种子在不同章节里**应该生长出不同的含义**

**对 LLM-as-judge 自查清单的新增项**：

- "本章复现了哪些种子？复现位置和原种子是否同源？"
- "新引入的种子是否 ≤ 1 个？"（一章最多新引入 1 颗新种子，剩下的配额都用于复现）

### 8.9 与项目原则的衔接

- 这套"具体性 × 主观性 × 密度 × 题材共鸣"原则（v3.1），是**产品级约束**——它不是 prompt 工程技巧，而是**生成器的审美宪法**
- 后续在 02 · 叙事引擎 里会转化为：
    - **四轴自查清单**（具体性 / 主观性 / 密度 / 题材共鸣）
    - **密度配比表**（按场景类型查）
    - **种子注册表**（跨章复现 2-3 颗种子）
    - **颗粒数/千字硬上限**（≤ 3 / 千字）
    - **LLM-as-judge**（每章生成后用另一个 LLM 评分，低于阈值则重新生成）
- **题材共鸣**前置到"故事开场前"——用户在故事开始前就要给出"我不爱什么"
