## v0.1 自评

### 跑通的部分
- **机制本身能跑** — 7 件产物 → 5 章 → 5 指标 → 报告，全链路打通
- **02 §2 权重公式** 数值正确（已和 02 §2.7 示例对比）
- **三层 prompt 组装** 结构正确（基座/动态/最近）
- **5 个指标计算器** 函数工作正常

### 发现的真问题（mock 数据限制）
- **5 章太短** — overdue 种子根本来不及触发（5 段 < 任何种子的 planned_interval）
- **mock 摘要简单** — 只检测关键词，不解析 LLM 输出 JSON
- **重要性 0.5** — 1/2 主线种子被强化，主要因为只有 1 个种子的 status 变了

### v0.2 要解决的
1. **接真实 friday 代理** 调真实 LLM（生成 + LLM-as-judge）
2. **扩到 20 章** 让 overdue 能触发
3. **解析 LLM 真实输出的 JSON** 替换 mock_extract_summary
4. **加 A/B 对照** 不同 02 参数下的指标差异
